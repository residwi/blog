# Personal Blog

A clean personal blog built with Jekyll and TailwindCSS v4, featuring a dedicated TIL (Today I Learned) section.

## Features

- **Jekyll 4.4.1** static site generator
- **TailwindCSS v4** with CSS-based configuration (`assets/css/application.css`)
- **Catppuccin** color palette (Latte light / Mocha dark)
- **Client-Side Search** powered by Lunr.js (loaded from CDN)
- **TIL Collection** as a separate Jekyll collection
- **Tag Pages** auto-generated by a custom Jekyll plugin
- **Code Syntax Highlighting** via Rouge with Catppuccin syntax themes
- **SEO** via `jekyll-seo-tag` and `jekyll-sitemap`, RSS via `jekyll-feed`

## Prerequisites

- **Ruby** 3.x with **Bundler**
- **Bun** (the project uses `bun.lock`)

## Local Development

### 1. Install dependencies

```bash
bundle install
bun install
```

### 2. Build CSS once

```bash
bun run build:css
```

### 3. Run the site

```bash
bun run dev
```

This builds the CSS once and starts `bundle exec jekyll serve --livereload`.

For a CSS watcher in a separate terminal:

```bash
bun run watch:css
```

Visit <http://localhost:4000>.

> Changes to `_config.yml` require a server restart — Jekyll does not reload it automatically.

## Available Scripts

| Script              | What it does                                                 |
| ------------------- | ------------------------------------------------------------ |
| `bun run build:css` | Compile `assets/css/application.css` → `assets/css/main.css` |
| `bun run watch:css` | Same as `build:css` but in `--watch` mode                    |
| `bun run dev`       | Build CSS once, then `bundle exec jekyll serve --livereload` |

## Creating Content

### Blog Posts

Create `_posts/YYYY-MM-DD-title.md`:

```markdown
---
title: "Your Post Title"
date: 2025-12-07 10:00:00 +0000
tags: [jekyll, tailwindcss]
---

Your content here...
```

The `post` layout and `/posts/:title` permalink are applied automatically via `_config.yml` defaults.

### TIL Entries

Create `_til/YYYY-MM-DD-title.md`:

```markdown
---
title: "What You Learned"
date: 2025-12-07
tags: [ruby]
---

Your quick note here...
```

The `til` layout is applied automatically.

### Tags

Tag pages are generated automatically at `/tags/<slug>/` by `_plugins/tag_generator.rb`, aggregating both posts and TIL entries that share a tag.

### Projects (optional)

The `/projects` page is hidden from the nav unless `show_projects: true` in `_config.yml`. Define projects in either:

- `_data/projects.yml`, or
- a top-level `projects:` key in `_config.yml`

Each project supports `name`, `description`, `tech`, `url`, `github`.

## Theming

Color theme uses [Catppuccin](https://catppuccin.com) (Latte / Mocha). Tailwind v4 configuration lives entirely in `assets/css/application.css` (no `tailwind.config.js`):

- `@import "tailwindcss"` — framework
- `@import "@catppuccin/tailwindcss/mocha.css"` — Catppuccin tokens
- `@plugin "@tailwindcss/typography"` — prose styles
- `@custom-variant dark (...)` — `dark:` variant maps to `.mocha`/`.frappe`/`.macchiato`
- `@theme { ... }` — fonts and animations

The theme toggle:

- An inline script in `_includes/head.html` sets `.mocha` or `.latte` on `<html>` before paint to avoid FOUC.

## Search

- `search.json` is rendered by Jekyll into a flat list of posts + TIL with `title`, `url`, `content`, `date`, `tags`, `type`.
- `assets/js/search.js` builds a Lunr index on first open of the search modal and runs queries client-side.
- Lunr is loaded from CDN (`unpkg.com/lunr@2.3.9`).

## Building

```bash
bun run build:css
bundle exec jekyll build
```

Output goes to `_site/`.
